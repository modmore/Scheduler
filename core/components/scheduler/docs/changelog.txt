Scheduler 1.9.0-pl
------------------
Released on 2026-01-22

- Make sTaskRun constants public, and standardise code style
- Add $task_key property to the sTask::schedule method to more easily set that [#40]
- Add recurring tasks support: tasks can automatically reschedule after successful execution
- Add `scheduleNext()` method to sTask for automatic rescheduling
- Fix race condition in run.php with atomic status update
- Add retry logic for failed tasks (max_retries, retry_delay fields)
- Add composite index (status, timing) for optimal task selection
- Fix time rounding - no longer rounds up when seconds = 0
- Add customizable email templates via scheduler.email_failure_tpl setting
- Add customizable email subject via scheduler.email_failure_subject setting
- Add scheduler.tasks_per_run setting to system settings
- Improve exception handling with Throwable (catches both Exception and Error)
- Add forward-compatible support for MODX 3 namespaced processors
- Add retry_count column to history grid
- Add max_retries and retry_delay fields to task edit window
- Memory cleanup after each task execution
- Add correct icon to menu

Notes on versions:
- The past few years, @biz87 has maintained a different version of Scheduler on their own, and released versions of Scheduler under the same name in the modstore package manager.
- On modstore, there were releases for a v1.5 on 2023-12-17, a v1.6 on 2025-01-16, v1.7 on 2024-12-01, and v1.8 on 2025-12-30.
- On modmore, there was a v1.5 release on 2025-02-14 that included some of the patches that @biz87 provided, but not all of them.
- v1.9 includes all the changes of both forks up to 2026-01-20.

Scheduler 1.3.0-pl
------------------
Released on 2021-02-20

- Add indices for various fields used for searching and filtering, should improve performance on sites with a large history substantially [#25]
- Add task_key field to sTaskRun, to allow setting/searching/indexing of small pieces of data. Among other things, this allows outside extras to retrieve and act on task runs more easily.
- Track the processing time of task runs [#20]
- Add optional automatic cleanup of old tasks. Set the scheduler.delete_tasks_after system setting to a value like "-1 year" or "-2 weeks" to remove completed and failed tasks older than that.
